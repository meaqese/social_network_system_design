# social network system design

## Функциональные требование:

- публикация постов из путешествий с фотографиями, небольшим описанием и привязкой к конкретному месту путешествия;
- лайки и комментарии постов других путешественников;
- подписка на других путешественников, чтобы следить за их активностью;
- поиск популярных мест для путешествий и просмотр постов с этих мест;
- просмотр ленты других путешественников и ленты пользователя, основанной на подписках в обратном хронологическом порядке;
- ленты есть в двух местах: лента постов и лента мест, оба выдают страницу с бесконечной пагинацией и 20 элементами;

## Нефункциональные требования:

- DAU 10 000 000 (за год, дальше будет еще больше)
- Аудитория СНГ, язык русский
- Высокая надежность (MTBF = 2200 часов)
- Удобство взаимодействия с мобильными устройствами и браузером. Мобильный веб зачастую неудобен, поэтому необходима реализация и мобильных приложений и веб приложения.
- Данные храним всегда (еще не видел соцсетей где удалялись данные), старые данные можно переводить в более дешевое и менее быстрые хранилища
- Доступность 99.95%
- Сезонность - летом, в новогодние, майские праздники - 2x
- Лимиты
	- максимальное количество подписчиков у пользователя - 5m
	- максимальное количество подписок у пользователя - 10k
	- максимум 50 подписок в день, чтобы меньше накрутки было
	- максимум 1 фотография в посте
	- максимальный размер текста в посте 150 символов
	- максимум 24 поста в день
	- максимальная длина комментария 150 символов
	- максимум 50 лайков в день
	- максимум 30 комментов в день
	- максимум 100 поисков в день
- Тайминги
	- публикация - 3 сек при 10 Мб/c у клиента
	- поиск - 2 сек при 10 Мб/c у клиента
	- загрузка ленты - 3 сек Мб/c у клиента
- Активность пользователей:
	- Добавление постов (в среднем) (write):
		- 1% пользователей добавляют 1 пост в день = 1% x 1 = 0.01
		- 5% пользователей 1 пост в 3 дня = 5% x (1 / 3) = 0.0166
		- 14% пользователей 1 пост в 10 дней = 14% x (1/10) = 0.014
		- остальные 80% пользователей добавляют 1 пост за 6 мес.  =~ 80% x (1/180) = 0.0044
		- суммарно = 0.01 + 0.0166 + 0.014 + 0.0044 = 0.045 поста в день от каждого пользователя = 450k постов в день
	- Просмотр ленты (read):
		- В среднем 40% пользователей просматривают ленту по 3 страниц в день, в странице 20 постов = 40% х (3х20) = 24
		- В среднем 60% пользователей просматривают 1 страницу в день (20 постов) = 12
		- Суммарно:
			- (40% x 3) + (60% x 1) = 1.8 запросов страницы на каждого пользователя в день
			- 24 + 12 = 36 постов в день в среднем каждый пользователь
	- Лайки, комменты и подписки (write):
		- средняя конверсия лайков - 10% = (10m * 36) х 10% = 36m лайков суммарно в день
		- средняя конверсия комментов - 0.5% = (10m * 36) x 0.5% = 1.8m комментов суммарно в день
		- средняя конверсия подписок - 0.1% = (10m * 36) x 0.1% = 360k подписок в день
	- Поиски мест (read):
		- в среднем 25% пользователей пользуются поиском. Один раз в неделю.
		- 30% пользователей которые нашли место, остаются смотреть ленту, в среднем 20 постов.
		- поиски - 25% x 10m x (1/7) =~ 355k поисков в день суммарно
		- проваливание в просмотр ленты - 30% x 355k x 1 страница (20 постов же) = 106k в день запросов страниц (+ к обычным просмотрам)

## Модель данных:

Пост ~ 0.5KB:
- id - 8B
- user_id - 8B
- place_id - 8B
- description - 400B
- created_at - 8B

Фото ~ 0.1KB
- id - 8B
- post_id - 8B
- url - 100B

Лайки ~ 24B
- id - 8B
- post_id - 8B
- user_id - 8B

Подписки ~ 16B
- followee_id - 8B
- follower_id - 8B

Комментарии ~ 132B
- id - 8B
- post_id - 8B
- user_id - 8B
- text - 100B
- created_at - 8B

Место ~ 138B
- id - 8B
- name - 30B
- url - 100B



## Расчеты RPS:
- Write
	- Добавление постов = 450k постов в день / 86400 = 5.2 RPS
	- Лайки = 36m per day / 86400 = 417 RPS
	- Комменты = 1.8m per day / 86400 = 21 RPS
	- Подписки = 360k per day / 86400 = 4 RPS
- Read
	- Просмотр ленты = 10m x 1.8 = 18m / 86400 = 208 RPS
	- Поиски мест = 355k / 86400 = 4 RPS
	- Лента мест = 106k / 86400 = 1.2 RPS

## Расчеты трафа:
### Метаданные:
- Write
	- Добавление постов = 0.5KB * 5.2 = 2.6KB/s
	- Лайки = 24B x 417 RPS = 10KB/s
	- Комменты = 132B x 21 RPS = 2.7KB/s
	- Подписки = 16B x 4 RPS = 64B/s
- Read
	- Просмотр ленты = 0.5KB x 20 x 208 =~ 2MB/s
	- Поиск мест (input) = 30B x 4  = 120B
	- Лента мест = 138B x 20 x 1.2 = 3.2KB

## Медиа:
В среднем фото поста/места весит 300КБ, при добавлении сжимается на клиенте

- Write
  - Добавление поста = 300KB * 5.2 RPS = 1.5MB/s
- Read
  - Просмотр ленты = 20 * 300KB * 208 = 1.2GB/s
  - Просмотр ленты мест = 20 * 300KB * 1.2 = 7MB/s
